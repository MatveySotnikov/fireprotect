# FireProtect Roof Calculator

## Техническое задание

### 1. ВВЕДЕНИЕ
Настоящее техническое задание (ТЗ) определяет требования к разработке микросервисного веб-приложения **«FireProtect Roof Calculator»** (далее — Система), предназначенного для автоматизированного расчета массы и объема огнезащитного состава для обработки кровельных конструкций в соответствии с методиками, изложенными в нормативных документах (ГОСТ Р 53292-2009).

Область применения: проектные организации, подрядчики по огнезащите, специалисты по пожарной безопасности.

### 2. ОСНОВАНИЯ ДЛЯ РАЗРАБОТКИ
*   **Документ-основание:** Учебный план дисциплины «Технологии создания программного обесечения».
*   **Организация:** РТУ МИРЭА.

### 3. НАЗНАЧЕНИЕ РАЗРАБОТКИ
**Функциональное назначение:** автоматизация рутинных расчетов огнезащиты для исключения ошибок ручного счета и хранения истории вычислений.
**Эксплуатационное назначение:** предоставление веб-интерфейса и API для интеграции в процессы проектирования и документооборота (формирование PDF-актов).

### 4. ТРЕБОВАНИЯ К ПРОГРАММЕ

#### 4.1. Требования к функциональным характеристикам
Система должна обеспечивать:
*   **Регистрация / Аутентификация:** по email и паролю с выдачей JWT-токенов.
*   **Справочник материалов:** хранение и публичный доступ к характеристикам огнезащитных составов (плотность, расход на I/II группу, коэффициенты потерь).
*   **Выполнение расчета:**
    *   Учет геометрии кровли: пересчет площади горизонтальной проекции через cos(α).
    *   Учет группы огнезащитной эффективности (I или II).
    *   Учет метода нанесения (кисть, распыление) с соответствующими коэффициентами потерь.
    *   Два режима ввода: из справочника или ручной ввод расхода/плотности.
*   **История расчетов:** сохранение всех результатов в базе данных с привязкой к пользователю.
*   **Генерация PDF-акта:** формирование отчета с результатами расчета.

#### 4.2. Требования к надежности
*   Отказоустойчивость внутренних сервисов (механизм retry для gRPC-вызовов).
*   Стабильная работа при запуске через Docker Compose с healthcheck-ами.

#### 4.3. Условия эксплуатации
*   Серверная часть: операционные системы, поддерживающие Docker Engine.
*   Клиентская часть: современные браузеры (Chrome, Firefox, Edge последних версий).

#### 4.4. Требования к составу и параметрам технических средств
*   Минимальные требования: 2 ГБ ОЗУ, 10 ГБ дискового пространства.
*   Должны быть установлены Docker, Docker Compose, Node.js (v18+).

#### 4.5. Требования к информационной и программной совместимости
*   Взаимодействие между сервисами: gRPC (protobuf), REST (JSON).
*   СУБД: PostgreSQL.
*   Языки: Go 1.22 (бэкенд), TypeScript (фронтенд).

## Описание программы

### 1. ОБЩИЕ СВЕДЕНИЯ
*   **Обозначение и наименование:** `fireprotect-roof-calculator` — комплекс микросервисов для расчета огнезащиты кровли.
*   **Программное обеспечение:** Docker Engine, Docker Compose, Node.js v18+, веб-браузер.
*   **Языки программирования:** Go 1.22, TypeScript (Vue 3), Protocol Buffers (gRPC).

### 2. ФУНКЦИОНАЛЬНОЕ НАЗНАЧЕНИЕ
Программа решает задачи классификации и расчета потребности в огнезащитных материалах (класс задач — инженерные расчеты). Функциональные ограничения: расчет ведется только для деревянных конструкций (расширение на металл запланировано).

### 3. ОПИСАНИЕ ЛОГИЧЕСКОЙ СТРУКТУРЫ
*   **Архитектура:** микросервисная.
*   **Компоненты:**
    *   **Gateway (REST API, порт 8080):** маршрутизация, JWT-авторизация, вызов gRPC-методов.
    *   **Calculator (gRPC, порт 50051):** stateless-сервис расчета массы и объема.
    *   **PdfService (gRPC, порт 50052):** генерация PDF-актов.
    *   **PostgreSQL:** хранение данных.
*   **Связи:** Gateway ↔ PostgreSQL (GORM), Gateway → Calculator/PdfService (gRPC с retry), Frontend → Gateway (REST).
*   **Алгоритм расчета:**
    1. Приведение площади (`S_факт = S_проекц / cos(α)`).
    2. Расчет итогового расхода (`R_итог = R_баз × K_потерь`).
    3. Вычисление массы (`M = S_факт × R_итог`) и объема (`V = M / ρ`).

### 4. ИСПОЛЬЗУЕМЫЕ ТЕХНИЧЕСКИЕ СРЕДСТВА
*   **Тип ЭВМ:** ПК с архитектурой x86-64.
*   **Устройства:** клавиатура, мышь, сетевая карта (для доступа по HTTP).

### 5. ВЫЗОВ И ЗАГРУЗКА
*   **Способ вызова серверной части:** `docker-compose up --build` из корня проекта.
*   **Способ вызова клиентской части:** `npm run dev` из папки `services/frontend`.
*   **Входная точка в программу:** веб-интерфейс доступен по адресу `http://localhost:5173`.

### 6. ВХОДНЫЕ ДАННЫЕ
*   **Характер данных:** текстовая информация (JSON), вводимая пользователем через веб-форму или отправляемая через REST API.
*   **Формат входного сообщения (`POST /calc`):**
    *   `area` (float64) — площадь (м²).
    *   `area_type` (string) — `"projection"` или `"slope"`.
    *   `slope_angle` (float64) — угол наклона (градусы).
    *   `target_group` (string) — `"1_group"` или `"2_group"`.
    *   `application_method` (string) — `"brush"`, `"spray_indoor"`, `"spray_outdoor"`.
    *   `material_id` (int, опционально) — ID из справочника.
    *   `normative_rate`, `density` (float64, опционально, при ручном вводе).

### 7. ВЫХОДНЫЕ ДАННЫЕ
*   **Характер данных:** JSON-ответ или бинарный файл (PDF).
*   **Формат выходного сообщения (`POST /calc` Response):**
    *   `total_mass` (float64) — масса состава (кг).
    *   `total_volume` (float64) — объем состава (л).
*   **Формат PDF-акта:** документ с полями ввода, результатами расчета и данными пользователя.
